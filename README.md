## Запуск проекта

```bash
npm install
npm run dev
```
## Общее описание бизнес-логики
Приложение загружает пользователей из JSON через fetch и отображает их на странице списка. Список пользователей представлен в виде таблицы и поддерживает:

1)поиск;
2)сортировку;
3)фильтрацию;
4)добавление пользователя;
5)удаление пользователя.

Для имитации работы с сервером добавлена асинхронная загрузка с loader.
После первого получения данных изменения сохраняются в localStorage, поэтому добавление и удаление не теряются после перезагрузки страницы. Страница групп формируется на основе текущего списка пользователей.
Группы собираются динамически: для каждой группы считается количество пользователей, а также отдельно учитываются пользователи без группы.

## Выводы по UI
Users page
Страница списка пользователей сделана в более строгом и простом стиле.
Основной акцент был на понятной структуре, таблице, фильтрах и рабочих сценариях.
Для такого экрана ручное проектирование UI подходит лучше, потому что дает больше контроля над удобством.
Groups page
Страница групп сделана более визуальной и карточной.
Для нее использовался LLM-подход: он помог быстрее собрать более выразительную композицию, анимации и обзорные блоки.
Общий вывод
Ручной подход лучше подходит для утилитарных экранов, где важны ясность и контроль.
LLM-подход удобен для более презентационных экранов, где нужно быстрее найти интересное визуальное решение.
